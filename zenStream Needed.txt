1. new dynamic FFMPEG commands to convert all sorts of media files to a conceivable format for streaming
 - Conceivable in the sense that, the format shall be HLS, and each stream shall have only stereo audio, and if no stereo audio is found,
    the convert the same audio to stereo, and if no audio is found, add a silent audio track to the stream.
 - FFMPEG commands shall be dynamic in the sense that, they shall be heavily dependent on the input media file, its no of audio streams
    and the no of video streams, and the codecs used in the media file, and the container format of the media file.
 - If a Video file includes Multiple audio streams, then the system shall generate multiple HLS streams, each with a different audio stream, and the same video stream, and the same subtitles if any.
    and the system shall generate a JSON Manifest or library to keep track of the multiple audio files available, and the master m3u8(playlist) file
    for each audio stream. subtitle streams if any shall be also noted and kept track in the JSON Manifest.
2. The JSON Manifest
    - The JSON Manifest shall be generated for all media files converted by the system to HLS
    - The JSON Manifest shall be of the following structure or shall closely resemble the following structure:
    {
        "movie_title": "The Movie Title",
        "movie_id": "unique_movie_id",
        "num_streams_a": 2,
        "num_streams_v": 1,
        "num_streams_s": 1,
        "streams": [
            {
                "streamType": "video",
                "streamId": "unique_stream_id",
                "codec": "h264",
                "resolution": "1920x1080",
                "playlist": "master.m3u8 || path_to_video_stream_playlist.m3u8"
            },
            {
                "streamType": "audio",
                "streamId": "unique_stream_id",
                "streamName": "English Audio Stream",
                "codec": "aac",
                "channels": 2,
                "language": "en",
                "playlist": "path_to_audio_stream_playlist.m3u8"
            },
            {
                "streamType": "audio",
                "streamId": "unique_stream_id",
                "streamName": "Spanish Audio Stream",
                "codec": "aac",
                "channels": 2,
                "language": "es",
                "playlist": "path_to_audio_stream_playlist.m3u8"
            },

            {
                "streamType": "subtitle",
                "streamId": "unique_stream_id",
                "streamName": "English Subtitle Stream",
                "codec": "webvtt",
                "language": "en",
                "path" : "path_to_subtitle_file.vtt"
            }
        ]
    }

    - here, Movie Title, Movie ID can be obtained from the database (the Movie ID shall be the unique ID provided by the TMDB // that is inturn stored in the db)
    - num_streams_a, num_streams_v, num_streams_s can be obtained from the media file using FFMPEG commands
    - streamType can be either video, audio, or subtitle
    - streamId can be generated using a unique ID generator (like UUID)
    - streamName is optional for Video streams, but for audio and subtitle streams, it can be obtained from the media file using FFMPEG commands (like the language of the audio stream, or the name of the subtitle stream)
    - codec can be obtained from the media file using FFMPEG commands
    - resolution can be obtained from the media file using FFMPEG commands (for video streams only)
    - channels can be obtained from the media file using FFMPEG commands (for audio streams only)
    - language can be obtained from the media file using FFMPEG commands (for audio and subtitle streams only)
    - playlist can be the path to the master m3u8 file for the video stream, or the path to the audio stream playlist for audio streams, or the path to the subtitle stream playlist for subtitle streams.
        the playlist path shall be the path of the HLS (m3u8) file generated by the FFMPEG command for that particular stream.
3. System Methodology for handling media,
    The ZenStream system already includes a certain methodology for splicing and converting media files to HLS format, but the new methodology shall be as follows:
    - The System shall slit up the input video file into bits/ or chunks, the larger the better, but the size of the chunks shall be determined by the system based on the input media file and its properties (like its duration, its bitrate, etc)
    - The System shall then convert each of these large chunk into smaller appropriate chunks of 10-15 seconds each, and each chunk shall be converted to HLS format, and the system shall generate a playlist (m3u8) file for each stream (video, audio, subtitle) for each chunk.
    - IF the audio stream is not stereo, the system shall convert it to stereo, and if no audio stream is found, the system shall add a silent audio track to the stream.
        In the case of the audio stream being converted to stereo, the system shall also make sure that the 7.1 / 5.1 / Dolby Atmos (whatever the original audio stream is) shall be downmixed to stereo in the the best possible/ appropriate way, with adjusting the position
        and such of the audio channels to make sure that the stereo audio output is of the be best possible quality and experience for the user.
    - THE DYNAMIC FFMPEG COMMANDS system is of utmost importance in this new methodology, as it shall be responsible for determining the appropriate FFMPEG command to use for each media file, 
    based on its properties and the desired output format (HLS with stereo audio).
4. File System After Conversion::
    - The File System or Folder Structure after conversion shall be as follows:
    /TMDB_ID/hls_output/
        - master.m3u8 (the master playlist for the video stream)
        - video_stream.m3u8 (the playlist for the video stream)
        - audio_stream_1.m3u8 (the playlist for the first audio stream)
        - audio_stream_2.m3u8 (the playlist for the second audio stream, if any)
        - Subtitle_Files/ (folder containing the subtitle files in .vtt format)
            - subtitle1.vtt
            - subtitle2.vtt
            - ...
        - video_chunks/ (folder containing the video chunks in .ts format)
            - chunk1.ts
            - chunk2.ts
            - ...
        - audio_chunks_1/ (folder containing the audio chunks for the first audio stream in .ts format)
            - chunk1.ts
            - chunk2.ts
            - ...
        - audio_chunks_2/ (folder containing the audio chunks for the second audio stream in .ts format, if any)
            - chunk1.ts
            - chunk2.ts
            - ...
        - video_manifest.json (the JSON Manifest file for the media file, containing all the information about the streams and their properties)
